# ECC Decoder Metrics

## Purpose

This note defines the decoder metrics used by the SPEC-v5 baseline artifact. It is research context for tests and claims; it is not a source of candidate LER values.

`artifacts/baselines/ler_baselines.json` contains only seeded MWPM baselines measured from the checked Stim manifest. It must not contain candidate decoder numbers, proxy formulas, neural-reference guesses, or dashboard values.

## Metrics Used By X9

| Metric | Definition | Use |
| --- | --- | --- |
| Logical error rate (LER) | `logical_errors / shots` after decoding sampled syndrome shots and comparing predicted observables to sampled logical observables. | Primary accuracy axis for baseline rows. |
| MWPM baseline | PyMatching minimum-weight perfect matching decode on Stim surface-code detector samples. | Zero-suppression anchor for future measured candidates. |
| Suppression vs MWPM | `(mwpm_ler - candidate_ler) / mwpm_ler`, only after a real candidate measurement exists. | Candidate comparison metric, not used to fabricate baselines. |
| Reproducibility epsilon | `1.96 * sqrt(p * (1 - p) / shots)` for each baseline LER estimate, with artifact-level epsilon set to the max row value. | Approximate 95% binomial sampling tolerance for reruns. |
| Provenance | Git SHA, branch, dirty flag, Stim manifest checksum, manifest file SHA-256, toolchain versions, split seed, scenario. | Makes baseline drift reviewable. |

## Primary Sources

### Surface-Code Decoder Target

Google DeepMind and Google Quantum AI describe AlphaQubit as a recurrent transformer-based neural decoder for surface-code experiments. The Nature paper defines the surface-code structure, detection events, logical observables, and code distance, then reports AlphaQubit improvements on Sycamore data and simulated distances up to 11. It also states the practical throughput target for superconducting qubits is 1 microsecond per round and that AlphaQubit remained slower than that target at publication time.

Source: [Learning high-accuracy error decoding for quantum processors, Nature 2024](https://www.nature.com/articles/s41586-024-08148-8)

Use in this repo: AlphaQubit is a research benchmark and claim anchor. Its reported neural-decoder performance must not be copied into `ler_baselines.json`; CryoBrain baseline rows are measured locally from Stim/PyMatching only.

### Stim Sampling Grounding

Stim is the checked simulator basis for the baseline rows. The Quantum paper presents Stim as a fast stabilizer-circuit simulator for QEC workloads and reports high-throughput sampling for large surface-code circuits.

Source: [Stim: a fast stabilizer circuit simulator, Quantum 5, 497 (2021)](https://quantum-journal.org/papers/q-2021-07-06-497/)

Use in this repo: `scripts/compute_ler_baselines_wsl.sh` uses `stim.Circuit.generated("surface_code:rotated_memory_z", ...)` and seeded detector sampling from the manifest scenario.

### MWPM Baseline Grounding

PyMatching is the local MWPM implementation used for baseline decoding. Its upstream README describes it as a Python/C++ QEC decoding library using minimum-weight perfect matching, and notes its integration with Stim for circuit-level noise sampling.

Source: [PyMatching upstream repository](https://github.com/oscarhiggott/PyMatching)

Use in this repo: `scripts/compute_ler_baselines_wsl.sh` decodes detector samples with `pymatching.Matching.from_stim_circuit` through the existing `decode_with_mwpm` helper.

## Baseline Artifact Rules

1. The JSON rows are split-level baselines from `tasks/cryo_brain_decoder/stim/manifest.json`.
2. Every row records the manifest split seed and the scenario actually sampled.
3. `manifest_vectors` is kept as manifest provenance for RTL stimulus alignment; LER uses `scenario.shots`.
4. The artifact-level epsilon is the maximum per-row `epsilon_95`.
5. Future candidate artifacts may compare against these MWPM rows, but must be generated by measured RTL or real model inference. They must not reuse formula LER or dashboard proxy outputs.
