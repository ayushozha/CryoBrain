# async_fifo (GEN second target)

The non-decoder RTL target that proves the CryoBrain measured optimization env
generalizes (SPEC-v5 §6 GEN gate). A two-lane round-robin stream-arbiter FIFO,
optimized for **measured sustained throughput** via the same
propose -> generate -> measure -> score -> memory loop as the decoder.

## Layout

| Path | Role |
|------|------|
| `rtl/stream_arb_fifo.sv` | Agent-facing starter (shallow depth-2 FIFO) — generated by `cryobrain.rtl_gen.fifo_generator`. |
| `fifo_config.json` | The starter `FifoConfig` (`depth`, `width`). |
| `scenario.json` | Traffic + metric config (cycles, seed, baseline depth). |
| `dv/cocotb/test_fifo_throughput.py` | Throughput DV: drives traffic, counts drained items, checks bit-exactness vs golden ref. Emits `FIFO_THROUGHPUT drained=.. cycles=.. match=..`. |
| `scripts/run_fifo_cocotb.py` | cocotb+Verilator runner (WSL). |
| `synth/synth.ys` | Yosys sign-off (area/cell stats). |
| `donotaccess/grade.py` | Hidden measured grader (throughput, correctness-gated). |
| `donotaccess/stream_arb_fifo_golden.sv` | Hidden deep reference (throughput ceiling oracle). |

## How it plugs into the env

- Generator: `cryobrain.rtl_gen.fifo_generator.generate_fifo_rtl(FifoConfig, out_dir)`
- Stimulus: `cryobrain.stim.fifo_stim.generate_traffic(...)` + `FifoRefModel`
- Measure: `cryobrain.accuracy.fifo_throughput.measure_fifo(rtl, depth=..)` (real sim)
- Loop: `cryobrain.rl.fifo_loop.run_fifo_training(steps=N)` — records variants in
  the SAME `cryobrain.memory.store.MemoryStore` as the decoder.

## Running

```bash
# Windows unit (sim boundary mocked):
PYTHONPATH=. python -m pytest tests/test_gen_fifo.py -q

# Real GEN gate (Linux/WSL, needs Verilator):
bash scripts/run_gen_fifo_wsl.sh 6
```
